
# Coconut Steering Experiment

## File Structure & Functionality Reference

### Overview

This document describes every file in the `steering/` experiment directory. The experiment tests two hypotheses about activation steering in Coconut (continuous-thought) models trained on ProsQA:

- **H1:** Intermediate latent stages (L2–L4) are immune to steering because Coconut encodes a superposition of next steps at those positions. A single directional vector cannot perturb a distributed representation.
- **H2:** The first and last (L1 and L5) latent steps are high-leverage injection points because L1 initializes reasoning and L5 commits to an answer path.

### File Details

#### `steering_config.yaml`

Central configuration consumed by all Python scripts. Keeps hyperparameters out of source code so runs are reproducible from the config alone.

**Key fields:**
- `model_path`: Path to trained Coconut checkpoint
- `val_path`: Path to `prosqa_valid.json`
- `train_path`: Path to `prosqa_train.json` (used for vector extraction)
- `layer`: Transformer layer index to hook into (default: 6, GPT-2 small has layers 0-11)
- `latent_positions`: List of positions to test: `[1, 2, 3, 4, 5]`
- `alpha_sweep`: List of steering strengths: `[5, 10, 20, 40]`
- `n_contrast`: Number of correct/wrong pairs for vector extraction
- `output_dir`: Directory for `results.csv` and `figures/`

*Note: Change `layer` to probe different depths of the residual stream.*

#### `extract_vectors.py`

Builds one steering vector per latent position by computing a contrastive mean difference in the residual stream. For each sample in the training set, the script runs two forward passes:

1.  **Correct path:** The sample as-is, where the model reasons toward `target`.
2.  **Counterfactual path:** The sample with `target` swapped for `neg_target`, forcing the model down the wrong branch.

At the specified transformer layer and latent token index, the hidden state is extracted from both passes. The steering vector for position `Li` is calculated as:

```
v_i = mean(correct hidden states at Li) - mean(wrong hidden states at Li)
```

Vectors are saved to `steering_vectors.pt` as a dictionary keyed by position index. This file is consumed by `run_steering_eval.py` and the analysis scripts.

*ProsQA fields used:* `steps` (to identify how many latent tokens a sample has), `target` (correct path), `neg_target` (counterfactual), and `idx_to_symbol` (for answer options).

#### `run_steering_eval.py`

The main evaluation loop. For each combination of (`latent_position`, `alpha`), it injects `alpha * v_i` into the residual stream at the latent token position, then runs inference over the full validation set.

For each sample the script records:
- `position`: Which latent step was perturbed (1–5, or 0 for baseline)
- `alpha`: Steering strength used
- `correct`: `1` if model predicted the expected answer, `0` otherwise
- `predicted_word`: The actual answer word the model output
- `expected_word`: The ground truth answer
- `n_steps`: Number of reasoning steps in this sample

Results are appended to `output_dir/results.csv`. A baseline row (`alpha=0`, no injection) is included for every sample so effect sizes are computed relative to unperturbed accuracy.

#### `analyze_steering_vectors.py`

Three-part analysis to validate whether steering vectors are meaningful:

1.  **Directionality check:** Cosine similarity between steering vector and (correct - wrong) differences on held-out test set.
2.  **PCA projection:** Projects correct/wrong thought representations into 2D, overlays steering vector direction, colors train vs test separately.
3.  **Norm diagnostics:** Compares thought vector norms to steering vector norms to explain why high alpha values are needed.

Outputs `figures/pca_L*.png` for each latent position.

#### `steering_analysis.py`

Advanced analyses for the paper/experiment:

1.  **Logistic regression:** Tests whether correct vs wrong representations are linearly separable at each stage.
2.  **Word2Vec-style analogy plots:** Projects onto steering direction (X-axis) and semantic residual (Y-axis). If the vector generalizes, correct-wrong pairs should show consistent horizontal offsets regardless of vertical (semantic) position.
3.  **Norm diagnosis table:** Summarizes vector norms, thought norms, required alpha for 1:1 perturbation, and separability accuracy.

Outputs `figures/analogy_L*.png` and prints summary tables to console.

### Execution Order

Run the scripts in this order for a complete experiment:

```bash
# 1. Extract steering vectors from the trained model
python steering/extract_vectors.py --config steering/steering_config.yaml

# 2. Run steering evaluation over the validation set
python steering/run_steering_eval.py --config steering/steering_config.yaml

# 3. Run analyses (can run independently after vectors exist)
python steering/steering_analysis.py --config steering/steering_config.yaml
python steering/analyze_steering_vectors.py --config steering/steering_config.yaml
```

### Dependencies

All scripts require:
- Python 3.8+
- PyTorch
- Transformers (Hugging Face)
- NumPy, Matplotlib
- scikit-learn
- pandas
- PyYAML

Install with:
```bash
pip install torch transformers numpy matplotlib scikit-learn pandas pyyaml
```

### Notes

- All scripts expect to be run from `~/coconut-cot/` (the project root)
- The `coconut` module must be in the Python path (handled by sys.path.insert in each script)
- `analyze_steering_vectors.py` requires the missing `from coconut import Coconut` import (add after line 60)
- `debug_steering.py` uses hardcoded paths - update before use if directory structure changes
